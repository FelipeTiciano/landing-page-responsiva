## Objetivo

Este projeto é a entrega final da disciplina, consolidando todo o conhecimento adquirido em desenvolvimento front-end, com foco em:

- Controle de versão profissional usando GitFlow e commits semânticos.
- Acessibilidade conforme WCAG 2.1 Nível AA.
- Otimização do projeto para produção.
- Documentação técnica completa.

---

## Tecnologias Utilizadas

- **HTML5** — estrutura semântica e compatibilidade com leitores de tela.
- **CSS3 / Flexbox / Grid** — layout responsivo, modo escuro e alto contraste.
- **JavaScript (Vanilla)** — interatividade e manipulação de elementos.
- **Git / GitHub** — versionamento, branches, commits semânticos e releases.
- **GitHub Pages** — deploy do site em ambiente de produção.
- **Imagens otimizadas** — compressão e placeholders ilustrativos.

## Instalação e Execução Local

1. Clone o repositório ou extraia o ZIP do projeto.
2. Abra a pasta no VS Code ou outro editor.
3. Abra `index.html` no navegador para visualizar o site.

Branches principais criadas:

- **main** — branch de produção
- develop** — branch de desenvolvimento (commit mínimo apenas para evidenciar o GitFlow)
- **feature/acessibilidade** — melhorias de acessibilidade (commit mínimo para evidenciar GitFlow)
- **feature/imagens** — branch criada para evidenciar o GitFlow (commit mínimo, sem alterações reais no projeto)
- **release/1.0.0** — preparação da versão final (commit mínimo, sem alterações reais no projeto)

Tag de release: v1.0.0

Todos os commits foram feitos com mensagens semânticas (feat:, fix:, chore:, merge:).
